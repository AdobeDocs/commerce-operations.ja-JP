# 変更を移行

増分移行ツールは、データ [&#128279;](data.md)の移行中に、Magento 1 データベースに（プレフィックス `m2_cl_*`を含む）差分テーブルと（変更履歴を追跡するための）トリガーをインストールします。 これらの差分テーブルとトリガーは、データの前回の移行以降にMagento 1で行われた変更のみを移行するために不可欠です。 変更点は次のとおりです。

* 顧客がストアフロントで追加したデータ（作成された注文、レビュー、顧客プロファイルの変更）

* 管理パネルの注文、製品、カテゴリのすべての操作

>[!NOTE]
>
>管理者を通じて入力されたその他の新規または更新されたエンティティ（属性ページやCMS ページなど）は、増分移行に含まれず、移行されません。


開始する前に、次の手順を実行して準備します。

1. アプリケーションサーバーに[&#x200B; ファイルシステム所有者](../../../installation/prerequisites/file-system/overview.md)としてログインします。
1. `/bin` ディレクトリに変更するか、システム `PATH`に追加してください。

詳しくは、[最初の手順](overview.md#first-steps) セクションを参照してください。

## 増分移行コマンドの実行

増分変更の移行を開始するには、次を実行します。

```shell
bin/magento migrate:delta [-r|--reset] [-a|--auto] {<path to config.xml>}
```

どこで：

* `[-r|--reset]`は、最初から移行を開始するオプションの引数です。 この引数は、移行のテストに使用できます。

* `[-a|--auto]`は、整合性チェック エラーが発生した場合に移行が停止するのを防ぐオプションの引数です。

* `{<path to config.xml>}`は`config.xml`への絶対ファイルシステムパスです。この引数は必須です。

>[!NOTE]
>
>増分移行は継続的なプロセスです。5秒ごとに自動的に再起動します。 移行プロセスを中止するには、CTRL-Cを使用します。


## サードパーティの拡張機能によって作成されたデータを移行する

`Delta` モードでは、[!DNL Data Migration Tool]は、Magentoの独自のモジュールのみが作成したデータを移行します。サードパーティのデベロッパーが作成したコードや拡張機能については責任を負いません。 これらの拡張機能がストアフロントデータベースでデータを作成し、マーチャントがMagento 2でこのデータを取得する場合は、それに応じて[!DNL Data Migration Tool]の設定ファイルを作成して変更する必要があります。

拡張機能に独自のテーブルがあり、差分移行のために変更内容を追跡する必要がある場合は、次の手順に従います。

1. 追跡するテーブルを`deltalog.xml` ファイルに追加します
1. `Migration\App\Step\AbstractDelta`を拡張する追加の差分クラスを作成します
1. 新しく作成したクラスの名前を`config.xml`のデルタ モード セクションに追加します
