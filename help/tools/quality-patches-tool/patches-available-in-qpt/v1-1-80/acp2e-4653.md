# ACP2E-4653: カート価格ルール カテゴリ スコープの条件が[!DNL REST] APIにありません

ACP2E-4653 パッチは、[!DNL REST] APIを通じてルールを取得または更新する際に、**[!UICONTROL Category (Parent Only)]**&#x200B;および&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;のカート価格ルール条件属性の範囲が公開されない問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80がインストールされている場合に利用できます。 パッチ IDはACP2E-4653です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p2

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

[!DNL REST] APIを通じてカート価格ルールを取得または更新すると、**[!UICONTROL Category (Parent Only)]**&#x200B;と&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;の属性範囲が公開されません。

<u>複製する手順</u>:

1. 管理者資格情報を使用して`POST` リクエストを`rest/V1/integration/admin/token`に送信して、管理トークンを生成します。
1. [!DNL REST] エンドポイント `rest/V1/salesRules`と`category_ids`属性を使用するアクション条件を使用して、カート価格ルールを作成します。
1. 管理パネルで、買い物かごの価格ルールを開きます。 **[!UICONTROL Actions]** セクションの&#x200B;**[!UICONTROL Apply the rule only to cart items matching the following conditions (leave blank for all items]**&#x200B;の下で、**[!UICONTROL Category (Children Only)]**&#x200B;条件を手動で追加します。
1. 条件を設定して、**[!UICONTROL Category (Children Only)]**&#x200B;が`<category_id>`にならないようにします。
1. `GET` リクエストを`rest/V1/salesRules/<rule_id>`に送信して、APIを介してルールを取得します。
1. 応答で返された`action_condition`値を確認します。

<u>期待される結果</u>:

1. [!DNL REST] APIを通じて、**[!UICONTROL Category (Children Only)]**&#x200B;条件を含むカート価格ルールを作成、取得、更新できます。
1. カテゴリ、**[!UICONTROL Category (Parent Only)]**&#x200B;および&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;を正しく区別できるように、属性スコープを明示的に公開する必要があります。 例えば、`action_condition`は次のように返されます。

   ```
           "action_condition": {
               "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product\\Combine",
               "conditions": [
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "!=",
                       "attribute_name": "category_ids",
                       "value": "4",
                       "extension_attributes": {
                           "attribute_scope": ""
                       }
                   },
                   {
                       "condition_type": "Magento\\SalesRule\\Model\\Rule\\Condition\\Product",
                       "operator": "==",
                       "attribute_name": "category_ids",
                       "value": "5",
                       "extension_attributes": {
                           "attribute_scope": "children"
                       }
                   }
               ],
               "aggregator_type": "all",
               "operator": null,
               "value": "1"
           },
   ```

<u>実際の結果</u>:

1. 両方の条件属性名は`category_ids`として返されます。
1. [!DNL REST] APIを通じて&#x200B;**[!UICONTROL Category (Children Only)]**&#x200B;条件を持つルールを作成または更新する方法が明確ではありません。
