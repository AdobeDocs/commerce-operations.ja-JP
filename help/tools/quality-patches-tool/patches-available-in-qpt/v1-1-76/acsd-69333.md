# ACSD-69333：予定されているアクティブな更新を含む製品で、SKU の変更が許可されていました

ACSD-69333 パッチでは、スケジュールされたアクティブなアップデートが含まれている製品の SKU の変更が許可されている問題が修正されています。 修正後は、スケジュールされたアクティブな更新の間、SKU の変更は禁止されます。 エラーが発生して製品の保存が失敗し、製品 SKU フィールドが [!UICONTROL Admin] ントロールパネルで編集できません。 これにより、ステージングロールバック中に SKU の変更によって発生する MSI インベントリの不整合を防止できます。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 がインストールされている場合に使用できます。 パッチ ID は ACSD-69333 です。 この問題はAdobe Commerce 2.4.9 で修正される予定であることに注意してください。

## 影響を受ける製品とバージョン

**Adobe Commerce バージョン用のパッチが作成されます。**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p6

**Adobe Commerce バージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7 ～ 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい [!DNL Quality Patches Tool] リリースを含む他のバージョンにも適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html) で互換性を確認します。 パッチ ID を検索キーワードとして使用して、パッチを見つけます。

## 問題

スケジュールされたアクティブな更新を含む製品の SKU が変更されました。 修正後は、スケジュールされたアクティブな更新の間、SKU の変更は禁止されます。 エラーが発生して製品の保存が失敗し、製品 SKU フィールドが [!UICONTROL Admin] ントロールパネルで編集できません。 これにより、ステージングロールバック中に SKU の変更によって発生する MSI インベントリの不整合を防止できます。

<u> 前提条件 </u>:

インベントリモジュールがインストールされ、有効になります。

<u> 再現手順 </u>:

1. [!UICONTROL Admin] パネルにログインします。
1. 新しいソースと新しい在庫を作成します。
1. シンプルな製品を作成します。 （例：SKU = *SP1*）
1. ステージング更新を作成します。 （例：名前を変更する）。
1. ステージングアップデートがアクティブな状態で、製品 SKU = *SP11* を変更します。
1. `inventory_source_item` テーブルを確認します。 新しい SKU = *SP11* で別のレコードが作成されます。
1. 使用可能なすべての数量を含んだ注文を行い、商品を **在庫切れ** にします。
1. 請求書と出荷を作成し、`inventory_source_item` テーブルを確認します。
1. SKU が *SP1* に戻り、在庫が **在庫中** であることを確認します。
1. ストアフロントに移動し、商品の在庫ステータスを確認します。

<u> 期待される結果 </u>:

商品の在庫ステータスは変更されません。

<u> 実際の結果 </u>:

在庫数が更新され、商品が **在庫** に移動されます。
