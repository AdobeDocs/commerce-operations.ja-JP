# 概要：[!DNL Quality Patches Tool] （QPT） v1.1.76

このサブセクションでは、[!DNL Quality Patches Tool] （QPT） v1.1.76 で使用可能なパッチによって修正された問題について詳しく説明します。

QPT v1.1.76 には、次のパッチが含まれています。
1. **ACSD-67091**：データ量に基づいて 2 つの削除戦略を実装することで、カタログルール製品インデックスのクリーンアップを確実に行うために、最大書き込みセットサイズのエラーを修正しました。
1. **ACSD-67370**:PDP/PLP のバンドル製品と、複数通貨店舗の買い物かごページで誤った価格が表示される複数の問題を修正しました。
1. **ACSD-68410**：交渉可能な見積もりに対して注文を行うと、追加の買い物かご行が見積もりに誤って追加または結合される問題を修正しました。 交渉可能な見積もりチェックアウトの最後の手順を終了した後、製品が買い物かごに正しく追加されるようになりました。
1. **ACSD-69086**:cron ジョブで変更ログテーブルをクリアできず、大量のデータを処理する際に [!DNL Galera Cluster] がクラッシュする問題を修正しました。
1. **ACSD-69115**：デフォルト以外の web サイトに割り当てられた顧客の買い物かごを管理する際に、管理者ユーザーに買い物かごのエラーが表示されなかった問題を修正しました。
1. **ACSD-69129**：デフォルトのベース Web サイトを削除し、セカンダリ Web サイトをデフォルトとして使用すると、[!DNL REST] API を使用してセカンダリ Web サイトの階層価格を更新しようとするとエラーが発生する問題を修正しました。
1. **ACSD-69203**：カテゴリ条件に複数のカテゴリがリストされている場合に、**[!UICONTROL Products List]** ウィジェットが誤った結果を返す問題を修正しました。
1. **ACSD-69261**：部分請求および残りの数量キャンセル シナリオでの `times_used` 属性の処理が誤っているので、顧客ごとの 1 回限りの使用に設定された買い物かご価格ルールクーポンが複数回再利用されていた問題を修正しました。
1. **ACSD-69308**:`special_price` が web サイトレベルでのみ設定された場合（**[!UICONTROL All Store Views]** で設定されていない場合）に、カタログ価格ルールが適用されない問題を修正しました。 修正後、カタログの価格ルールは、最初に web サイトのデフォルトストアを確認することで、正しく適用されます。
1. **ACSD-69319**：子製品がカスタムソースで在庫されていた場合に、バンドル価格のインデックスが正しく作成されなかった問題を修正しました。
1. **ACSD-69325**:SKU のケースを変更すると、ストアフロントで製品が在庫切れに表示される問題を修正しました。
1. **ACSD-69331**：メディアギャラリーのコンテンツ作成者が `create_folder` 権限のみを使用してフォルダーを作成できなかった問題を修正しました。 修正後、期待どおりにフォルダーを作成できます。
1. **ACSD-69333**：スケジュールされたアクティブなアップデートを含む製品で SKU の変更が許可されていた問題を修正しました。 修正後は、アクティブな更新中に SKU の変更が禁止され、保存が失敗して明確なエラーが表示され、管理者の SKU フィールドが無効になります。 これにより、MSI が妨げられます。
1. **ACSD-69541**：管理画面に表示される商品の数量を、買い物かごに既に存在する数量より少なくすると、GraphQLを使用してその買い物かごの商品数量を編集できなくなる問題を修正しました。

左側のメニューを使用して、特定のパッチページに移動します。
