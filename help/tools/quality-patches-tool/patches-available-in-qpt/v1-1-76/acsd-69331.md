# ACSD-69331:**[!UICONTROL Create Folder]** 権限のみを持つ管理者ユーザーの場合、メディアギャラリーフォルダーの作成が失敗する

ACSD-69331 パッチは、**[!UICONTROL Create Folder]** アクセス権のみが付与されている場合に、管理者ユーザーがCMS Media Gallery にフォルダーを作成できなかった問題を修正しています。 修正後、期待どおりにフォルダーを作成できます。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76 がインストールされている場合に使用できます。 パッチ ID は ACSD-69331 です。 この問題はAdobe Commerce 2.4.9 で修正される予定であることに注意してください。

## 影響を受ける製品とバージョン

**Adobe Commerce バージョン用のパッチが作成されます。**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p8

**Adobe Commerce バージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.5 ～ 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい [!DNL Quality Patches Tool] リリースを含む他のバージョンにも適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html) で互換性を確認します。 パッチ ID を検索キーワードとして使用して、パッチを見つけます。

## 問題

Media Gallery の **[!UICONTROL Create Folder]** 権限のみを持つ管理者ユーザーは、**[!UICONTROL Delete Folder]** 権限も付与されていない限り、フォルダーを作成できません。

<u> 再現手順 </u>:

1. 管理パネルにログインします。
1. **[!UICONTROL Dashboard]** および **[!UICONTROL Content]**/**[!UICONTROL Elements]**/**[!UICONTROL Media Gallery]** へのアクセス権を持つ新しい役割を作成し、**[!UICONTROL Create Folder]** 権限のみを付与します（**[!UICONTROL Delete Folder]** は付与しません）。
1. 新しい管理者ユーザーを作成し、この役割を割り当てます。
1. 新しい管理者ユーザーとしてログインします。
1. **[!UICONTROL Content]**/*[!UICONTROL Media]*/**[!UICONTROL Media Gallery]** に移動します。
1. ディレクトリを選択します（例：`catalog`）。
1. 「**[!UICONTROL Create Folder]**」をクリックします。
1. フォルダー名を入力して確認します。

<u> 期待される結果 </u>:

* フォルダーの作成は、**[!UICONTROL Create Folder]** 権限のみを持つ管理者ユーザーに対して機能します。
* フォルダーを作成するために、**[!UICONTROL Delete Folder]** 権限は必要ありません。

<u> 実際の結果 </u>:

* ユーザーは、**[!UICONTROL Create Folder]** 権限のみを持つフォルダーを作成できません。
* フォルダーを作成するには、**[!UICONTROL Delete Folder]** 権限の付与が必要です。
