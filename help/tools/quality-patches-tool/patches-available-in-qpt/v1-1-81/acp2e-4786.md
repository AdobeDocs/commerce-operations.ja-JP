# ACP2E-4786:AWS S3 リモートストレージが設定されていると、製品の書き出しが失敗する

ACP2E-4786 パッチは、AWS S3 リモートストレージが有効になっている場合にストレージドライバーが無効なパスを使用するため、製品の書き出しが失敗する問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81がインストールされている場合に利用できます。 パッチ IDはACP2E-4786です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p4

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.5 - 2.4.9

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ja)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

ストレージドライバーでのパス処理が無効なため、AWS S3 リモートストレージが有効になっている場合、製品の書き出しが失敗します。

<u>複製する手順</u>:

1. [!DNL AWS S3]をリモート ストレージとして設定：

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. 次のコマンドを実行して、メディアファイルのデータベースストレージを無効にします。

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. 管理者サイドバーで、**[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **書き出し**&#x200B;に移動します。
1. **[!UICONTROL Entity Type]**&#x200B;に対して、**[!UICONTROL Products]**&#x200B;を選択します。
1. **[!UICONTROL Continue]**&#x200B;をクリックします。

<u>期待される結果</u>:

書き出しファイルが正常に作成されました。

<u>実際の結果</u>:

1. 書き出しファイルは作成されません。
1. ログに&#x200B;*SignatureDoesNotMatch* エラーが表示されます。
