# ACSD-68499: GraphQL updateCartItems mutationは、利用可能な在庫を超える数量について&#x200B;*success*&#x200B;を返します

ACSD-68499 パッチは、利用可能な在庫を超える数量を更新し、膨らんだ数量と合計が発生する場合に、GraphQL `updateCartItems`の突然変異が誤った&#x200B;*success*&#x200B;応答を返す問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74がインストールされている場合に利用できます。 パッチ IDはACSD-68499です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p2

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

Adobe CommerceのGraphQL `updateCartItems`のミューテーションは、利用可能な在庫を超える数量を更新する際に&#x200B;*success*&#x200B;の応答を誤って返し、カートの合計と数量が膨らみます。

<u>複製する手順</u>:

1. 空のカートを作成します。
1. 在庫が限られている商品（10個など）をカートに追加します。
1. 買い物かごにクエリを実行し、`cart_item_id`をメモします。
1. 次の突然変異を使用して、カートの品目数量を100 （有効在庫が10を超える）に更新します。

   ```graphql
   mutation ($cart: String!, $cartItemId: Int!, $quantity: Float!) {
     updateCartItems(
       input: {
         cart_id: $cart
         cart_items: [{ cart_item_id: $cartItemId, quantity: $quantity }]
       }
     ) {
       cart {
         items {
           product {
             name
           }
           quantity
         }
         prices {
           grand_total {
             value
             currency
           }
         }
       }
     }
   }
   ```

1. カートを再度問い合わせて、実際の数量を確認します。

<u>期待される結果</u>:

在庫検証は、無効な数量の更新を防ぎます。

<u>実際の結果</u>:

`updateCartItems`変異は、数量100の&#x200B;*成功*&#x200B;応答を返します。
