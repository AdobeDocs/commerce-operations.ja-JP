# ACSD-67643：高ネスト化されたカテゴリ環境で、スケジュールされた更新中に作成されたエントリが重複する

ACSD-67643 パッチは、ネストされたカテゴリの数が多い環境で、スケジュールされた更新中に重複するエントリが作成される問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.71がインストールされている場合に利用できます。 パッチ IDはACSD-67643です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p1

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8 - 2.4.8-p2

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

多数のネストされたカテゴリを持つ環境に対してスケジュールされた更新を作成すると、エントリが重複します。

<u>複製する手順</u>:

1. **サンプルデータを生成：**
修正を含むパフォーマンス修正ファイル （例：`small.xml`）を使用します。
   **[!UICONTROL Categories]**: *2,500*
   **[!UICONTROL Products]**: *250,000*

1. 次のコマンドを実行します。

   ```shell
   bin/magento setup:performance:generate-fixtures var/performance-toolkit/profiles/ce/small.xml
   ```

1. 製品とカテゴリを作成した後、次のコマンドを実行して、すべてのカテゴリがアンカーであることを確認します。

   ```sql
   UPDATE catalog_category_entity_int SET value = 1 WHERE attribute_id = (SELECT attribute_id FROM eav_attribute WHERE attribute_code = 'is_anchor');
   ```

1. 完全なインデックス再作成を実行します。
1. バックエンドにログインし、**[!UICONTROL Categories]** ページを開きます。
1. *カテゴリ 2*&#x200B;を&#x200B;*カテゴリ 1*&#x200B;に移動して、ツリーのネストを増やします。
1. 最もネストされたカテゴリに移動します（例：*カテゴリ 2.1.1*）。
1. **スケジュール済み更新**&#x200B;を作成し、カテゴリを保存します。

<u>期待される結果</u>:

カテゴリが正常に保存されました。

<u>実際の結果</u>:

*エラー500*&#x200B;が発生し、次のメッセージがログに表示されます。

```text
    Magento\Framework\DB\Adapter\DuplicateException:
    SQLSTATE[23000]: Integrity constraint violation: 1062 Duplicate entry '6-165' for key 'PRIMARY', 
    query was: INSERT INTO `temp_category_descendants_68a35aea273fc` (`category_id`, `descendant_id`)
    SELECT `ce`.`entity_id` AS `category_id`, `ce2`.`entity_id` AS `descendant_id`
    FROM `catalog_category_entity` AS `ce`
    INNER JOIN `catalog_category_entity` AS `ce2` ON ce2.path LIKE CONCAT(ce.path, '/%')
```
