# ACSD-68206: [!DNL GraphQL] サーバーがレート制限とPHP Redis拡張機能で失敗します

ACSD-68206 パッチは、レート制限機能とPHP Redis拡張機能を使用する際にGraphQL アプリケーションサーバーがエラーをスローし、レート制限機能が中断する問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73がインストールされている場合に利用できます。 パッチ IDはACSD-68206です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p2

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

この問題は、GraphQL アプリケーションサーバーがPHP Redis拡張機能がインストールされたレート制限機能を使用し、レート制限機能が破損した場合に発生します。

<u>複製する手順</u>:

1. PHP Redis拡張機能をインストールします。
1. アプリケーション用にRedisを設定します。
1. レート制限を設定して、顧客ごとに1回の注文を毎分許可します。
1. [!DNL GraphQL] アプリケーションサーバーを起動します。
1. 顧客として、`PlaceOrder` [!DNL GraphQL]の変異を使用して注文を送信します。
1. 1分以内に、同じ顧客に`PlaceOrder` [!DNL GraphQL]の突然変異を使用して2回目の注文を送信します。

<u>期待される結果</u>:

`PlaceOrder` [!DNL GraphQL] リクエストは、注文の試行を制限するメッセージを返します。

<u>実際の結果</u>:

2回目のPlaceOrder GraphQL リクエストの場合、サーバーは500 エラーを返します。 例外ログレコード：

```text
report.CRITICAL: Error: Call to a member function exec() on null in vendor/colinmollenhour/credis/Client.php:1336
```
