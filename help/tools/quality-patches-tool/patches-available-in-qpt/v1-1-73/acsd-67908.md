# ACSD-67908：マルチストア設定でJavaScript ファイルが正しく結合されない

ACSD-67908 パッチは、マルチストア設定でJavaScript ファイルが正しく結合できない問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73がインストールされている場合に利用できます。 パッチ IDはACSD-67908です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

>[!NOTE]
>
>Adobeでは、**[!UICONTROL Merge JavaScript Files]**&#x200B;の使用はお勧めしません。 このパッチを適用する前に、JavaScriptの結合と静的コンテンツの再デプロイを無効にして、問題が解決されるかどうかを確認してください。 詳しくは、[&#x200B; リソースファイルの最適化](/help/implementation-playbook/best-practices/development/optimize-css-js-files.md#merge-js-files)を参照してください。


## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p5

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

結合されたJavaScript ファイルがマルチストア設定で正しく生成されず、ファイルが破損したり、ストアフロントのエラーが発生したりします。

<u>複製する手順</u>:

1. 複数のweb サイト、ストア、ストアビューを設定する。
1. 実稼動モードを設定し、コンパイルをスキップします。

   ```
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. 生成されたコードをデプロイします。

   ```
   bin/magento setup:di:compile
   ```

1. コンパクトな戦略で静的コンテンツをデプロイ：

   ```
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. 結合されたキャッシュフォルダーの内容を削除します。

   ```
   rm -rf pub/static/_cache/merged/*
   ```

1. キャッシュをクリア：

   ```
   bin/magento cache:clean
   ```

1. すべてのストアビューを別々のブラウザータブまたはウィンドウで同時に開きます。


<u>期待される結果</u>:

すべてのJavaScript ファイルが正常に結合され、ストアフロントはエラーなしで機能します。

<u>実際の結果</u>:

結合されたJavaScript ファイルの一部が破損しており、正しい[!DNL RequireJS] モジュール宣言で始まりません。
