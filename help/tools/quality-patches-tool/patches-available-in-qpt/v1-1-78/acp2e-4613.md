# ACP2E-4613：大きなメディアディレクトリが原因で、メディアギャラリーディレクトリツリーの読み込みが遅れる

ACP2E-4613 パッチは、大きなメディアディレクトリ構造が`gettree`応答を遅くし、**[!UICONTROL Media Gallery]** ディレクトリツリーの読み込み時間が長くなる問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78がインストールされている場合に利用できます。 パッチ IDはACP2E-4613です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p3

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

大きなメディアディレクトリ構造では、`gettree`応答が遅くなり、[!UICONTROL Media Gallery] ディレクトリツリーの読み込み時間が長くなります。

<u>前提条件</u>:

1. 複数の画像を持つ商品が多数存在します。

<u>複製する手順</u>:

1. 画像のサムネールを生成して、メディアフットプリントを拡大：

   ```
   bin/magento catalog:images:resize
   ```

1. `pub/media/catalog/category`の下に複数のサブディレクトリを作成します（例：test、test2、test3、test4、および必要に応じてネストされた追加のフォルダー）。
1. カタログメディアファイルの大規模なセットを新しく作成したディレクトリにコピーして、メディアディレクトリツリーのサイズを大幅に増やします。次に例を示します。

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. 管理者パネルにログインし、**[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**&#x200B;に移動します。
1. `admin/media_gallery/directories/gettree/?isAjax=true` リクエストの完了に時間がかかります。

<u>期待される結果</u>:

ディレクトリツリーは数秒以内に読み込まれます。

<u>実際の結果</u>:

ディレクトリツリーは、`pub/media`の下にあるサブディレクトリの数に応じて、読み込みに数分かかります。
