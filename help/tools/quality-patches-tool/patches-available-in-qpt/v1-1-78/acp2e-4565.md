# ACP2E-4565: X-Adobe-Company ヘッダーが使用されている場合、Company GraphQL クエリは&#x200B;*現在のお客様が許可されていません*&#x200B;を返します

ACP2E-4565 パッチは、指定された会社と店舗に正しく割り当てられていても、X-Adobe-Company ヘッダーとStore ヘッダーの両方がリクエストに含まれていると、会社GraphQL クエリがエラー&#x200B;*現在のお客様は許可されていません*&#x200B;を返す問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78がインストールされている場合に利用できます。 パッチ IDはACP2E-4565です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p3

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p3

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ja)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

指定された会社と店舗に適切に割り当てられているにもかかわらず、X-Adobe-Company ヘッダーとStore ヘッダーの両方がリクエストに含まれている場合、会社GraphQL クエリは&#x200B;*現在のお客様は許可されていません*&#x200B;を返します。

<u>複製する手順</u>:

1. B2B企業を支援する。
1. web サイト、ストア、ストアビューを追加作成する。
1. 2つの会社（会社1と会社2）を作成します。
1. 2番目のweb サイトに割り当てられたユーザーを作成します。
1. 両方の会社をユーザーに割り当てます。
1. これらのヘッダーを使用して、次のGraphQL クエリを実行します。
   * `X-Adobe-Company: MQ==`
   * `Store: storeview2`

   ```graphql
   query Company {
     company {
       id
       legal_name
       name
     }
   }
   ```

<u>期待される結果</u>:

企業データが正常に返されます。

<u>実際の結果</u>:

GraphQLの応答で、認証エラーが返されます。

```
{
    "errors": [
        {
            "message": "The current customer isn't authorized.",
            "extensions": {
                "category": "graphql-authorization"
            }
        }
    ]
}
```
