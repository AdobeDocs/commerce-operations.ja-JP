# ACP2E-4416：管理画面で作成した場合、顧客報酬ポイントが初期化されない

ACP2E-4416 パッチは、Adminで作成されたときに顧客報酬ポイントが初期化されない問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78がインストールされている場合に利用できます。 パッチ IDはACP2E-4416です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p3

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

管理画面で顧客報酬ポイントを作成しても、顧客報酬ポイントは初期化されません。

<u>複製する手順</u>:

1. AdminまたはREST APIを使用して顧客を作成します。
1. GraphQL リクエストを送信して、顧客の報酬ポイント残高を取得します。
1. 応答を確認します。

<u>期待される結果</u>:

システムは、顧客の報酬ポイント残高を返します。

<u>実際の結果</u>:

システムがエラーを返します。

```
{"errors":[{"message":"Internal server error","locations":[{"line":5,"column":9}],"path":["customer","reward_points","balance","points"]}],"data":{"customer":{"reward_points":{"balance":null}}}}
```
