# ACP2E-4555: Telephone numbers that contain periods or forward slashes don&#39;t validate properly

The ACP2E-4555 patch fixes the issue where modern telephone numbers containing `.` or `/` don&#39;t validate. このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78がインストールされている場合に利用できます。 The patch ID is ACP2E-4555. この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p3

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

Modern telephone numbers containing `.` or `/` don&#39;t validate.

<u>複製する手順</u>:

1. Log in as a customer on the storefront.
1. Add a new address to the address book with a telephone number, for example: `06.76.40.32.22` or `+43680/2149568`.
1. Save the address details.

<u>期待される結果</u>:

ピリオドやスラッシュを使った現代的な電話形式を使うことも可能です。

<u>実際の結果</u>:

`.`または`/`文字が原因で、電話フィールドの検証が失敗します。
