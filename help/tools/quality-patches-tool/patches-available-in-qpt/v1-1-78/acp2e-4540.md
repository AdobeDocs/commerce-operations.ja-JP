# ACP2E-4540: Fotorama ライブラリが正しく読み込まれないので、最初に添付された画像のみが表示されます

ACP2E-4540 パッチは、Fotorama ライブラリが正しく読み込まれず、最初に添付された画像のみが表示される問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78がインストールされている場合に利用できます。 パッチ IDはACP2E-4540です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7-p7

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

Fotorama ライブラリが正しく読み込まれず、最初に添付された画像のみが表示されます。

<u>複製する手順</u>:

1. シンプルな商品に複数の画像を添付。
1. ストアフロントで商品を開きます。

<u>期待される結果</u>:

画像ギャラリーには、すべての添付された画像が表示されます。

<u>実際の結果</u>:

1. 最初の画像のみが表示されます。
1. ブラウザーコンソールに次のエラーが表示されます。

```
Javascript error(severe) - http://magento.com/static/version1717863530/frontend/magento/luma/en_us/mage/gallery/gallery.js 309:29 uncaught type error: $fotorama element.fotorama is not a function
```

## パッチを適用する

個別のパッチを適用するには、デプロイメント方法に応じて次のリンクを使用します。

* Adobe CommerceまたはMagento Open Source オンプレミス：[!DNL Quality Patches Tool] ガイドの[[!DNL Quality Patches Tool] >使用状況](/help/tools/quality-patches-tool/usage.md)。
* クラウドインフラストラクチャ上のAdobe Commerce:「[ アップグレードとパッチ > パッチを適用](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/upgrade/apply-patches.html)」（Commerce クラウドインフラストラクチャガイド）。
