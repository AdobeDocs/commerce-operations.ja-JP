# ACSD-68892：キャッシュ可能なページ [!DNL Fastly] キャッシュ動作が一貫していない

ACSD-68892 パッチを使用すると、キャッシュ可能なページが [!DNL Fastly] キャッシュに適切に格納または提供されなくなり、キャッシュ動作の不一致とパフォーマンスの低下が発生する問題を修正できます。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77 がインストールされている場合に使用できます。 パッチ ID は ACSD-68892 です。 この問題はAdobe Commerce 2.4.9 で修正される予定であることに注意してください。

## 影響を受ける製品とバージョン

**Adobe Commerce バージョン用のパッチが作成されます。**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p2

**Adobe Commerce バージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい [!DNL Quality Patches Tool] リリースを含む他のバージョンにも適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html) で互換性を確認します。 パッチ ID を検索キーワードとして使用して、パッチを見つけます。

## 問題

キャッシュ可能なページが [!DNL Fastly] キャッシュに適切に格納されず、提供もされないため、キャッシュ動作に一貫性がなくなり、パフォーマンスが低下します。

<u> 再現手順 </u>:

1. [!DNL Fastly] キャッシュを有効にします。
1. 新しい web サイト/ストア/ストア表示を作成します。
1. セカンダリ Web サイトコードを `index.php` に追加します。

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. curl を使用してキャッシュコントロールヘッダーを確認します。

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u> 期待される結果 </u>:

キャッシュ可能なページは、[!DNL Fastly] にキャッシュされます。

<u> 実際の結果 </u>:

キャッシュ可能なページは [!DNL Fastly] にキャッシュされず、`max-age=0` はヘッダーで返されます。

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
