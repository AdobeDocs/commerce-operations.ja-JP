# ACSD-68892: キャッシュ可能なページの[!DNL Fastly] キャッシュ動作に一貫性がありません

ACSD-68892 パッチは、キャッシュ可能なページが[!DNL Fastly] キャッシュに適切に保存または提供されない問題を修正し、結果としてキャッシュ動作の一貫性が失われ、パフォーマンスが低下します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77がインストールされている場合に利用できます。 パッチ IDはACSD-68892です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8-p2

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ja)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

キャッシュ可能なページが[!DNL Fastly] キャッシュに適切に保存されていないか、サービスが提供されていないため、キャッシュ動作が一貫性がなく、パフォーマンスが低下します。

<u>複製する手順</u>:

1. [!DNL Fastly] キャッシュを有効にします。
1. 新しいweb サイト/ストア/ストアビューを作成します。
1. セカンダリ web サイト コードを`index.php`に追加します。

```shell
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. curlを使用してキャッシュコントロールヘッダーを確認します。

```shell
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>期待される結果</u>:

キャッシュ可能なページは[!DNL Fastly]にキャッシュされます。

<u>実際の結果</u>:

キャッシュ可能なページは[!DNL Fastly]にキャッシュされません。`max-age=0`はヘッダーに返されます。

```shell
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
