# ACSD-63687: [!DNL Redis] キャッシュのクリーンアップの問題により、誤った価格が表示される

ACSD-63687 パッチは、[!DNL Redis] キャッシュガベージコレクターがLUA モードなしで実行され、競合条件が発生した場合に誤った価格が表示される問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77がインストールされている場合に利用できます。 パッチ IDはACSD-63687です。 この問題は、Adobe Commerce 2.4.9で修正される予定です。

## 影響を受ける製品とバージョン

**パッチはAdobe Commerceのバージョン**&#x200B;用に作成されました

* Adobe Commerce（すべてのデプロイメント方法） 2.4.5-p9

**Adobe Commerceのバージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） 2.4.5 - 2.4.6-p13

>[!NOTE]
>
>このパッチは、新しい[!DNL Quality Patches Tool] リリースを含む他のバージョンに適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]：パッチの検索ページ ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)で互換性を確認します。 パッチ IDを検索キーワードとして使用して、パッチを検索します。

## イシュー

[!DNL Redis] キャッシュのクリーンアップが失敗するため、価格が正しく表示されません。

<u>複製する手順</u>:

1. Adobe Commerce環境用に[!DNL Redis]を設定します。
1. ストアフロントで複数の商品カテゴリーを作成する。
1. 定期的なリクエストをカテゴリーページに送信するスクリプトを作成します。 JMeter スクリプトまたは同等のロード テスト ツールを使用します。
1. スクリプトがカテゴリーページにリクエストを送信する間に、`block_html` キャッシュの削除を実行し、`backend_clean_cache` cron ジョブを複数回実行します。
1. 指定されたスクリプトを実行して、MAGE キャッシュセットに存在しない[!DNL Redis] キャッシュキーレコードを識別します。 [!DNL Redis] ポートとMAGE キャッシュ タグのプレフィックスを更新して、ローカル プロジェクト設定と一致させます。

   ```shell
   #!/bin/bash
   
   REDIS_PORT=6379
   REDIS_DB=0
   SET_NAME="zc:ti:af6_MAGE"
   
   redis-cli -p $REDIS_PORT -n $REDIS_DB keys "zc:k:*" | while read KEY; do
     STRIPPED_KEY=${KEY#"zc:k:"}
     EXISTS=$(redis-cli -p $REDIS_PORT -n $REDIS_DB sismember "$SET_NAME" "$STRIPPED_KEY")
   
     if [ "$EXISTS" -eq 0 ]; then
       TIMESTAMP=$(redis-cli -p $REDIS_PORT -n $REDIS_DB hget "$KEY" "m")
   
       if [ -n "$TIMESTAMP" ]; then
         FORMATTED_TIME=$(date -r "$TIMESTAMP" +"%y-%m-%d %H:%M:%S")
         echo "$KEY - m: $FORMATTED_TIME"
       else
         echo "$KEY - m: (null)"
       fi
     fi
   done
   ```

<u>期待される結果</u>:

対応するレコードのないキャッシュキーが[!DNL Redis]に作成されず、`backend_clean_cache` Cron ジョブがキャッシュエントリを正しくクリーニングします。

<u>実際の結果</u>:

対応するレコードを含まないキャッシュキーが[!DNL Redis]に作成されるため、`backend_clean_cache` cronによってトリガーされるキャッシュクリーンアップは機能しません。
