# ACSD-68451：会社管理者が、複数の web サイトをまたいで関連のない会社にリンクする

ACSD-68451 パッチは、会社の管理者がある web サイトにログインし、別の web サイトで無関係の会社を作成し、その無関係の会社と誤って関連付けられる問題を修正します。 このパッチは、[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75 がインストールされている場合に使用できます。 パッチ ID は ACSD-68451 です。 この問題は、Adobe Commerce [!DNL B2B] 1.5.3 で修正される予定です。

## 影響を受ける製品とバージョン

**Adobe Commerce バージョン用のパッチが作成されます。**

* Adobe Commerce（すべてのデプロイメント方法） – [!DNL B2B] 1.5.2-p2

**Adobe Commerce バージョンとの互換性：**

* Adobe Commerce（すべてのデプロイメント方法） – [!DNL B2B] 1.5.2-p1 - 1.5.2-p3

>[!NOTE]
>
>このパッチは、新しい [!DNL Quality Patches Tool] リリースを含む他のバージョンにも適用される可能性があります。 パッチがAdobe Commerceのバージョンと互換性があるかどうかを確認するには、`magento/quality-patches` パッケージを最新バージョンに更新し、[[!DNL Quality Patches Tool]: Search for patches page](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=ja) で互換性を確認します。 パッチ ID を検索キーワードとして使用して、パッチを見つけます。

## 問題

複数の web サイトを設定する場合、会社の管理者が 1 つの web サイトにログインして会社を作成します。 次に、同じブラウザーで、管理者が別の web サイトを開き、別の会社を作成します。 会社は無関係であるにもかかわらず、プラットフォームは 2 番目の会社管理者を 1 番目の会社と誤って関連付けます。

<u> 前提条件 </u>:

[!DNL B2B] と Inventory をインストールします。

<u> 再現手順 </u>:

1. 追加の web サイト、ストア、ストア表示を作成します。
1. *[!UICONTROL Company]*/**[!UICONTROL Stores]**/**[!UICONTROL Configuration]**/**[!UICONTROL General]** の **[!UICONTROL B2B Features]** を有効にします。
1. ストアフロントのデフォルトの web サイトにアクセスします。
1. ストアフロントで新しい会社を作成し、会社管理者としてログインします。
1. 同じブラウザーセッションで、セカンダリ web サイトにアクセスします。
1. セカンダリ Web サイトのストアフロントに新しい会社を作成します。
1. 管理パネルにログインし、カスタマーグリッドを確認します。

<u> 期待される結果 </u>:

別の会社管理者が同じブラウザーの別の web サイトにログインしたままの場合でも、新しい会社管理者は、新しく作成した会社にのみ割り当てられます。

<u> 実際の結果 </u>:

新しい会社管理者は、対象となる会社に加えて、他の会社にも **会社ユーザー** として割り当てられます。
