# サンプルデータの概要

サンプルデータは、Luma テーマに基づいたストアフロントを提供します。これには、製品、カテゴリー、顧客登録などが含まれます。 Commerceのストアフロントと同様に機能し、管理者を使用して価格、在庫、プロモーションの価格設定を調整できます。

>[!NOTE]
>
>データベースや様々な機能を確認および分析するには、サンプルデータの代わりに実際のデータを使用することを検討してください。 サンプルデータは、事前に生成されたストアシミュレーションとして設計されており、テーマのデザインと基本的なストアフロントの動作を示します。 サンプルデータエンティティはすべてデータベーステーブルに直接書き込まれますが、サンプルデータはインストールされます。

サンプルデータは、Commerce ソフトウェアのインストール前またはインストール後にインストールできます。 サンプルデータの処理が完了したら、削除するか、[&#x200B; サンプルデータモジュールの削除またはサンプルデータの更新](remove-or-update.md)の説明に従って新しいデータをインストールします。

>[!WARNING]
>
>サンプルデータはアンインストールできません。 サンプルデータのみを使用して、Adobe Commerceの仕組みについて学習します。 サンプルデータをインストールしたシステムで開発を行うことは避けてください。

オプションのサンプルデータは、次のいずれかの方法でインストールできます。

| インストール方法 | 説明 | 必要なスキルレベル |
|--- |--- |--- |
| Composerの使用 | [`magento sampledata:deploy`を実行して、アプリケーションのルート `composer.json`](composer-packages.md)を変更し、サンプルデータモジュールを有効にします。 | Commerce ファイルシステムへのコンポーザーの知識とアクセスが必要です。 |
| リポジトリの複製 | [GitHub リポジトリ &#x200B;](git-repositories.md)とサンプル データ リポジトリを複製してから、それらをリンクします。 | 貢献開発者のみ。 他の人は前の方法のいずれかを使用する必要があります。 |
