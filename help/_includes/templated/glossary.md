# Adobe Commerce用語集

## A

### 一目瞭然の

_形容詞の_

ブラウザーウィンドウで、Web ページが読み込まれた直後、およびユーザーがページを下にスクロールした前に表示されるコンテンツ。
レイアウトをデザインする際には、柔軟な形式を使用して、この領域で最も優先度の高い製品、機能、販売、通知、オプションなどを最適に表示します。

モバイルとタブレットでは、折り目の上の領域が大きく異なります。特に、表示時の画面と向きのサイズとサイズ（縦置きか横置きか）によって異なります。
レスポンシブテーマとテストを使用すると、コンテンツとレイアウトの適切な組み合わせを見つけるのに役立ちます。

_用語の属性：_

* _フィールド：デザイン_

### 活動枝

_名詞_

アクティブなブランチまたは環境は、サービスへのアクセス権を持つデプロイ済みまたは実行中のインスタンスに接続されるブランチまたは環境です。
非アクティブ化すると、サービスと実行中のインスタンスへの接続は削除されますが、コードは保持されます。
通常の Git ブランチになります。

_用語の属性：_

* _フィールド： cloud_

### アダプタ

_名詞_

システムのソースコードを変更せずに、他の互換性のない 2 つのシステムを連携させることを可能にするクラス。
例としては、データベースアダプタ、キャッシュアダプタ、ファイルシステムアダプタ、ポストプロセッサライブラリアダプタ、その他の種類のコンピューティングアダプタなどがあります。

_用語の属性：_

* _フィールド：プログラミング_

### admin

_名詞_

ソフトウェアでは、すべての機能を管理するための完全な管理者権限を持つユーザーの役割です。
Adobe Commerceでは、管理者ユーザーは、管理者のすべての機能、オプションおよび機能に対するフル権限とアクセス権を持ちます。
また、ユーザーや役割を作成することもできます。

詳細情報： [ユーザーの追加](https://experienceleague.adobe.com/docs/commerce-admin/systems/user-accounts/permissions-users-all.html)

_用語の属性：_

* _フィールド：コマースソフトウェア_
* _シノニム： administrator、super user_
* _関連用語： commerce admin_

### 管理領域

_名詞_

ストアのパスワードで保護されたバックオフィス。注文、カタログ、コンテンツ、設定を管理します。
ユーザーは管理領域にアクセスして、製品、注文、出荷、CMS コンテンツ、ストアフロントのデザイン、顧客情報など、ストアを管理します。
管理者ユーザーには、機能、オプションおよび機能へのアクセスを制御する権限を持つロールが関連付けられています。

詳細情報： [Adobe Commerce User Guide](https://experienceleague.adobe.com/docs/commerce-admin/user-guides/home.html)

_用語の属性：_

* _フィールド：コマースソフトウェア_
* _シノニム：管理、管理パネル、バックエンド、管理インターフェイス、管理 UI_
* _関連用語： admin_

### 領域

_名詞_

領域は、Magento・アプリケーション・スコープの抽象用語です。
領域は、最適化されたリクエスト処理のためのコードを整理する論理コンポーネントです。
領域を使用すると、ストアフロントからアクセスする設定オブジェクトのメモリ要件が減り、必要な依存コードのみを読み込むことで、Web サービス呼び出しを合理化できます。
各領域には、URL とリクエストを処理するための完全に異なるコードを含めることができます。

Adobe Commerceの領域は次のとおりです。

* 管理者 (adminhtml)
* ストアフロント
* Web API REST(webapi_rest)

_用語の属性：_

* _フィールド：コマースソフトウェア_
* _関連用語：コマースコンポーネント、ストアフロント_

### 属性

_名詞_

製品のある側面を記述する製品の特性またはプロパティ。
Adobe CommerceまたはMagento Open Sourceのユーザーは、カスタム属性を作成して、デフォルトの属性セットまたはカスタム属性セットに追加できます。
管理者またはプログラムを使用して、これらの属性を作成します。
例：色、サイズ、重み付け、価格、年齢、性別など。

カスタム属性は、Entity-Attribute-Value(EAV) 属性の一種です。

Google Shopping Ads チャネルやAmazonSales Channelなどの統合の場合、コマース属性をサードパーティの属性にマッピングして、製品の適切な表示と販売、広告の表示をおこなうことができます。

詳細情報： [EAV と拡張機能の属性](https://developer.adobe.com/commerce/php/development/components/attributes/)

_用語の属性：_

* _フィールド：コマースソフトウェア、製品_
* _シノニム：製品属性、カスタム属性_
* _関連用語：拡張機能属性_

必要に応じて通貨記号をカスタマイズし、各店舗または表示ごとに価格の表示を個別に設定できます。

_名詞_

### カタログ

_名詞_

マーチャントの場合、カタログは商品の在庫を表します。
Adobe Commerceアーキテクチャでは、カタログは、カテゴリ、製品および製品属性で構成されます。

各コマースストアには 1 つの製品カタログのみがあり、すべてのストア表示で共有されます。
マルチストアインストールでは、各ストアに個別のカタログを割り当てたり、カタログを共有したりできます。
現在のストアカタログは、デフォルトのルートカテゴリによって決まります。このカテゴリは、ストアの上部のナビゲーション（メインメニュー）に表示されます。
（「ルートカテゴリ」という用語は、実際には「ルートカテゴリ」とは言えませんが、カテゴリとサブカテゴリで構成されるメニューのコンテナなので、混乱の元になる可能性があります）。

ルートカテゴリは必要な数だけ作成できますが、一度に 1 つ（デフォルト）のみ使用できます。

_用語の属性：_

* _フィールド：コマースソフトウェア、価格、製品_
* _関連用語：共有カタログ、カタログルール_

### 設定可能な製品

_名詞_

設定可能な製品は、各バリエーションのオプションのドロップダウンリストを含む単一の製品のように見えます。
各オプションは、実際には一意の SKU を持つ個別のシンプルな製品なので、各製品バリエーションの在庫を追跡できます。
同様の効果を実現するために、シンプルな製品をカスタムオプションで使用できますが、バリエーションごとに在庫を追跡する機能はありません。
複数のオプションを持つ製品は、複合製品と呼ばれる場合があります。

設定可能な製品ではより多くの SKU を使用し、最初は設定に少し時間がかかる場合がありますが、最終的に時間を節約できます。
ビジネスを成長させる予定がある場合は、複数のオプションを持つ製品に対して、設定可能な製品タイプが最適な選択肢となる場合があります。

例： T シャツは 2 色と 3 色のサイズで使用できます。
6 つのバリアントを個々の製品（それぞれ独自の SKU を持つ）として作成する必要があります。
次に、すべてのバリエーションが設定可能な製品に追加され、顧客がサイズと色を選択して、買い物かごに追加できます。

_用語の属性：_

* _フィールド：コマースソフトウェア、製品_
* _関連用語：製品タイプ_
