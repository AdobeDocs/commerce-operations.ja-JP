# パートナーのエスカレーションのベストプラクティス

以下は、Adobe アカウントチームとのパートナーエスカレーションに備えたり、エスカレーションを完全に回避したりするのに役立つアイデアです。

## 影響を受ける製品とバージョン

[ サポートされているすべてのバージョン ](../../../release/versions.md) /:

* Adobe Commerce on cloud infrastructure
* Adobe Commerce オンプレミス

## ベストプラクティス

それぞれの状況は異なり、以下のチェックリスト項目の一部には適用されない可能性があることを理解しています。 しかし、お客様がAdobeのアカウントチームに連絡を取ると、通常、そのプロセスが迅速化され、より迅速に解決できます。

* **スコープを検討**：成果物に満足していない場合、または成果物が満たされていないと思われる場合は、パートナーから提供されたスコープドキュメントを再検討することをお勧めします。 同様に、成果物が追加された場合は、それがタイムラインにどのような影響を与えるのかを検討します。
* **あなたの懸念をパートナーに伝えますか？** 課題に直接ソースで対処する方が良い場合もあります。 パートナーと懸念事項について話し合い、問題に対処する機会を提供しましたか？ 多くの場合、パートナーとできるだけオープンで透明性の高い関係を築くことで、プロジェクトにポジティブな変化がもたらされます。 パートナーの問題に対処しようとして、回答がない、または明確な回答がない場合は、Adobe アカウントチームと相談することをお勧めします。
* **具体的であること**：例を挙げてください。 これにより、電子メールやサポートチケットなどの詳細情報を短時間で確認できるようになりました。懸念事項について具体的な例を挙げれば挙げるほど、効果は高まります。
