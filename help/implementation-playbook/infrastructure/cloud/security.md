# セキュリティ

AdobeCommerce Pro プランのアーキテクチャは、高度に安全な環境を提供するように設計されています。 各顧客は、他の顧客とは別に、独自の独立したサーバ環境に導入されます。 実稼動環境のセキュリティの詳細については、以下で説明します。

## Web ブラウザー

クラウド環境で出入りするトラフィックの大部分は、消費者の Web ブラウザーから発生します。 消費者トラフィックは、Web サイト上のすべてのページで HTTPS を使用して保護できます ( 共有 SSL 証明書または顧客独自の SSL 証明書を使用します（追加料金）。 チェックアウトページとアカウントページは、常に HTTPS を使用して提供されます。 ベストプラクティスは、すべてのページを HTTPS で提供することです。

## コンテンツ配信ネットワーク (CDN)

Fastly は、CDN および分散型 DoS(DoS) 保護を提供します。 Fastly CDN は、オリジンサーバーへの直接アクセスを分離するのに役立ちます。 パブリック DNS は Fastly ネットワークのみを指しています。 Fastly DoS ソリューションは、レイヤ 3 およびレイヤ 4 の高度な破壊攻撃や、より複雑なレイヤ 7 攻撃から保護します。 レイヤ 7 攻撃は、HTTP/HTTPS 要求全体、およびヘッダー、Cookie、リクエストパス、クライアント IP などのクライアントとリクエスト条件、または位置情報などの指標に基づいて、カスタムルールを使用してブロックできます。

## Web アプリケーションファイアウォール (WAF)

Fastly Web Application Firewall（WAF；ファストリー Web アプリケーションファイアウォール）は、追加の保護を提供するために使用されます。 Fastly のクラウドベースの WAF は、OWASP Core Ruleset など、商用およびオープンソースのソースから提供されるサードパーティのルールを使用します。 また、Adobeコマース固有のルールが使用されます。 お客様は、インジェクション攻撃や悪意のある入力、クロスサイトスクリプティング、データ抽出、HTTP プロトコル違反、その他の OWASP Top 10 の脅威など、主要なアプリケーション層攻撃から保護されています。

WAF ルールは、ソフトウェアパッチの前にManaged Servicesがセキュリティの問題を「事実上パッチ」できるように、新しい脆弱性が検出された場合に、Adobeコマースによって更新されます。 Fastly WAF は、レート制限またはボット検出サービスを提供しません。 必要に応じて、Fastly と互換性のあるサードパーティのボット検出サービスのライセンスを取得できます。

## Virtual Private Cloud (VPC)

AdobeCommerce Pro プランの実稼動環境は、Virtual Private Cloud(VPC) として設定され、実稼動サーバーを分離して、クラウド環境との接続/接続/接続を制限できます。 セキュリティで保護されたクラウドサーバーへの接続のみが許可されます。 SFTP や rsync などのセキュアなプロトコルは、ファイル転送に使用できます。

お客様は、SSH トンネルを使用して、アプリケーションとの通信を保護できます。 AWS PrivateLink へのアクセスは追加料金で提供できます。 これらのサーバーへの接続はすべて、環境への接続を制限する仮想ファイアウォールである AWS セキュリティグループを使用して制御されます。 お客様の技術リソースは、SSH を使用してこれらのサーバーにアクセスできます。

## 暗号化

Amazon Elastic Block Store (EBS) は、ストレージに使用されます。 すべての EBS ボリュームは、AES-265 アルゴリズムを使用して暗号化されます。 つまり、保存時にデータが暗号化されます。 また、CDN とオリジン間、およびオリジンサーバー間のデータの転送も暗号化します。 顧客のパスワードはハッシュとして保存されます。 支払いゲートウェイ用の資格情報を含む機密性の高い資格情報は、SHA-256 アルゴリズムを使用して暗号化されます。

Adobeコマースアプリケーションは、データが保存されていない場合や、サーバー間で転送されていない場合は、列レベルまたは行レベルの暗号化や暗号化をサポートしません。 お客様は、アプリケーション内から暗号化キーを管理できます。 システムで使用されるキーは AWS 鍵管理システムに保存され、サービスの一部を提供するためにManaged Servicesで管理される必要があります。

## 侵入テスト

Managed Servicesは、標準のアプリケーションを使用して、AdobeCommerce cloud システムの定期的な侵入テストを実施します。 お客様は、カスタマイズされたアプリケーションの侵入テストを担当します。

## 支払ゲートウェイ

Adobeコマースでは、クレジットカードデータが消費者のブラウザーから支払ゲートウェイに直接渡される、支払ゲートウェイ統合が必要です。 カードデータは、Commerce Pro プランのどのAdobeでもManaged Services環境では使用できません。 e コマースアプリケーションによるトランザクションに対するアクションは、ゲートウェイからのトランザクションへの参照を使用して実行されます。

## Adobeコマースアプリケーション

Adobeは、コアアプリケーションコードを定期的にテストして、セキュリティの脆弱性を調べます。 欠陥およびセキュリティ問題に関するパッチは、お客様に提供されます。 製品セキュリティチームは、OWASP アプリケーションのセキュリティガイドラインに従って、Adobeコマース製品を検証します。 いくつかのセキュリティ脆弱性評価ツールと外部ベンダーを使用して、コンプライアンスのテストと検証を行います。 次のセキュリティツールが含まれます。

- Veracode 静的および動的スキャン
- RIPS ソースコードスキャン
- Trustwave と Alert Logic の脆弱性スキャンサービス
- バープスイートプロ
- OWASPZAP
- andSqlMap

完全なコードベースは、これらのツールを週 2 回スキャンします。 お客様には、直接の電子メール、アプリケーション内の通知、および [ セキュリティセンター ](https://magento.com/security) からセキュリティパッチが通知されます。

お客様は、PCI ガイドラインに従って、リリースから 30 日以内に、これらのパッチがカスタマイズされたアプリケーションに適用されていることを確認する必要があります。 Adobeには [ セキュリティスキャンツール ](https://docs.magento.com/user-guide/magento/security-scan.html) も用意されており、マーチャントは定期的にサイトを監視し、既知のセキュリティリスク、マルウェア、不正アクセスに関する更新を受け取ることができます。 セキュリティスキャンツールは無料のサービスで、任意のバージョンのAdobeコマースで実行できます。

セキュリティ研究者に脆弱性の特定と報告を促すため、Adobe・コマースには、内部テストに加え、[ バグバウンティ・プログラム ](https://hackerone.com/magento) が用意されています。 さらに、お客様は、必要に応じて、アプリケーションの完全なソースコードを自分のレビュー用に提供します。

## 読み取り専用ファイル・システム

すべての実行可能コードは読み取り専用のファイル・システム・イメージにデプロイされ、攻撃に使用できるサーフェスが大幅に削減されます。 展開プロセスで Squash-FS イメージが作成されます。 このアプローチは、PHP または JavaScript コードをシステムに挿入したり、Commerce アプリケーションファイルを変更したりする機会を大幅に減らします。

## リモートデプロイメント

実行可能コードをManaged Servicesの実稼動環境に取り込む唯一の方法は、プロビジョニングプロセスを通じて実行することです。 これには、ソースリポジトリからリモートリポジトリにソースコードをプッシュして、デプロイメントプロセスを開始する必要があります。 そのデプロイメントターゲットへのアクセスが制御されるので、デプロイメントターゲットにアクセスできるユーザーを完全に制御できます。 非実稼動環境へのアプリケーションコードのデプロイメントはすべて、お客様が管理します。

## ログ

すべての AWS アクティビティは、AWS CloudTrail に記録されます。 Linux、アプリケーション・サーバ、データベース・ログは、本番サーバに保存され、バックアップに保存されます。 ソースコードの変更はすべて Git リポジトリに記録されます。 展開履歴は、Adobeコマース [Project Web Interface](https://devdocs.magento.com/cloud/project/projects.html#login) で確認できます。 すべてのサポートアクセスが記録され、サポートセッションが記録されます。

## 機密データ

機密データは、消費者からの個人情報や、Managed Servicesのお客様からの機密データをカバーできます。 機密性の高い顧客および消費者データの保護は、AdobeCommerce Managed Servicesにとって重要な義務です。 Managed Servicesもお客様も、個人を特定できる情報に関する法的義務を負っています。 アーキテクチャのセキュリティ機能に加えて、機密データの配布とアクセスを制限する他のコントロールもあります。

お客様はデータを所有し、そのデータの場所を管理できます。 顧客は、実稼動インスタンスと開発インスタンスが存在する場所を指定します。 また、コマースと組み合わせてMagento Business Intelligence(MBI) 環境に使用する場所を指定し、その MBI アプリケーションが個人を特定できる情報にアクセスできるかどうかを指定します。 実稼動インスタンスはほとんどの AWS リージョンに配置できますが、開発環境と MBI 環境は、現時点では米国または欧州連合に配置できます。

機密データは、Fastly CDN サーバーネットワークを通過する場合がありますが、Fastly ネットワークには保存されません。 Managed ServicesAdobeに含まれるすべてのパートナーは、機密データの保護を確保する契約上の義務を負います。 Managed Servicesは、機密性の高い顧客データや消費者データを、顧客が指定した場所から移動することはありません。

Managed Servicesのスタッフは、AdobeコマースManaged Services製品に含まれるサービスを提供する一環として、機密データを含む実稼動システムにアクセスする必要があります。 これらの従業員は、機密性の高い顧客および消費者のデータを保護するための義務に基づいてトレーニングを受けます。 これらのシステムへのアクセスは、アクセスを必要とする従業員に限られます。 これらの従業員は、これらのサービスを提供するのに必要な時間だけアクセスできます。

## 一般データ保護規則 (GDPR)

GDPR は、欧州連合 (EU) 内の個人の個人情報の収集と処理に関するガイドラインを定めた法的枠組みです。 これらの規制は、サイトのベースとなる場所に関係なく適用され、EU 訪問者はサイトにアクセスできます。

基本的に、訪問者は、サイトで収集したデータを通知し、情報収集に明示的に同意する必要があります。 サイトが保持する個人データが漏洩した場合、サイトは訪問者に通知する必要があります。

また、サイトを運営する商人や会社には、サイトのデータセキュリティを監督する専門のデータ保護担当者が必要です。また、訪問者からデータの消去を要求された場合は、この個人（または Web サイト管理チーム）に問い合わせてください。

また、GDPR は、匿名化または偽名化された個人を特定できる情報（名前、人種、生年月日など）を収集する必要もあります。

## バックアップ

バックアップは、過去 24 時間の操作に対して 1 時間ごとに実行されます。 24 時間の後、AWS EBS Snapshot サービスを使用して、次のスケジュールでバックアップが保持されます。

| 期間 | バックアップ保存ポリシー |
|----------------|-------------------------|
| 1 ～ 3 日 | 各バックアップ |
| 4 ～ 6 日 | 1 日に 1 回のバックアップ |
| 第 2 週から 6 週 | 1 週間に 1 回のバックアップ |
| 週 8 ～ 12 | 隔週 1 回のバックアップ |
| 週 12 ～ 22 | 1 ヶ月に 1 回のバックアップ |

これにより、冗長ストレージに対して独立したバックアップが作成されます。 EBS ボリュームは暗号化されるので、バックアップも暗号化されます。 さらに、Managed Servicesは、お客様からリクエストされた方法でオンデマンドバックアップを実行します。

AdobeCommerce Managed Servicesのバックアップ/リカバリアプローチでは、高可用性アーキテクチャとフルシステムバックアップを組み合わせて使用します。 各プロジェクトは、3 つの異なる AWS アベイラビリティーゾーンにまたがって、すべてのデータ、コード、アセットをレプリケートします。各ゾーンは別々のデータセンターを持ちます。
