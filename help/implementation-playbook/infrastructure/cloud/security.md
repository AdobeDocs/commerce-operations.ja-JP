# セキュリティ

ザAdobe Commerce [プロプランアーキテクチャ](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/architecture/pro-architecture.html) は、安全性の高い環境を提供するように設計されています。 各顧客は、他の顧客とは別に、独自の独立したサーバ環境に導入されます。 実稼動環境のセキュリティの詳細については、以下で説明します。

## Web ブラウザー

クラウド環境で出入りするトラフィックの大部分は、消費者の Web ブラウザーが発生します。 消費者トラフィックは、Web サイト上のすべてのページで HTTPS を使用して保護できます（共有 SSL 証明書または追加料金で顧客独自の SSL 証明書を使用）。 チェックアウトページとアカウントページは、常に HTTPS を使用して提供されます。 ベストプラクティスは、HTTPS の下のすべてのページを提供することです。

## コンテンツ配信ネットワーク

Fastly は、コンテンツ配信ネットワーク (CDN) および分散型 DoS(DoS) 保護を提供します。 Fastly CDN は、オリジンサーバーへの直接アクセスを分離するのに役立ちます。 パブリック DNS は Fastly ネットワークのみを指しています。 Fastly DoS ソリューションは、非常に破壊的なレイヤ 3 およびレイヤ 4 攻撃や、より複雑なレイヤ 7 攻撃から保護します。 レイヤ 7 攻撃は、HTTP/HTTPS 要求全体に基づくカスタムルールと、ヘッダー、cookie、リクエストパス、クライアント IP などのクライアントとリクエストの条件に基づくカスタムルールを使用してブロックできます。

詳しくは、 [Fastly サービスの概要](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/cdn/fastly.html) （内） _クラウドガイド_.

## Web アプリケーションファイアウォール

Fastly Web Application Firewall（WAF; Fastly Web アプリケーションファイアウォール）は、追加の保護を提供するために使用されます。 Fastly のクラウドベースの WAF は、OWASP Core Ruleset などの商用およびオープンソースから提供されるサードパーティのルールを使用します。 また、Adobe Commerce固有の規則も用いられる。 お客様は、インジェクション攻撃や悪意のある入力、クロスサイトスクリプティング、データ抽出、HTTP プロトコル違反、その他の OWASP の上位 10 の脅威など、アプリケーション層の主要な攻撃から保護されています。

WAF ルールは、新しい脆弱性が検出された場合にAdobe Commerceによって更新され、Managed Servicesはソフトウェアパッチの前にセキュリティの問題を「事実上パッチ」できます。 Fastly WAF では、レート制限やボット検出サービスは提供されません。 必要に応じて、Fastly と互換性のあるサードパーティのボット検出サービスのライセンスを取得できます。

詳しくは、 [Web アプリケーションファイアウォール (WAF)](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/cdn/fastly-waf-service.html) （内） _クラウドガイド_.

## Virtual Private Cloud

Adobe Commerce Pro プランの実稼動環境は Virtual Private Cloud(VPC) として設定され、実稼動サーバは分離され、クラウド環境との接続/接続/接続が制限されます。 セキュリティで保護されたクラウドサーバーへの接続のみが許可されます。 SFTP や rsync などのセキュアなプロトコルをファイル転送に使用できます。

お客様は、SSH トンネルを使用して、アプリケーションとの通信を保護できます。 AWS PrivateLink へのアクセスは、追加料金で提供できます。 これらのサーバーへの接続はすべて、環境への接続を制限する仮想ファイアウォールであるAWSセキュリティグループを使用して制御されます。 お客様の技術リソースは、SSH を使用してこれらのサーバーにアクセスできます。

## 暗号化

Amazon Elastic Block Store(EBS) は、ストレージに使用されます。 すべての EBS ボリュームは、AES-256 アルゴリズムを使用して暗号化されます。つまり、保存時にデータが暗号化されます。 また、CDN とオリジン間、およびオリジンサーバー間の送信データも暗号化します。 顧客のパスワードはハッシュとして保存されます。 支払いゲートウェイの資格情報を含む機密性の高い資格情報は、SHA-256 アルゴリズムを使用して暗号化されます。

データが保存されていない場合や、サーバー間で転送されていない場合、Adobe Commerceアプリケーションは列レベルまたは行レベルの暗号化や暗号化をサポートしません。 お客様は、アプリケーション内から暗号化キーを管理できます。 システムで使用されるキーは、AWS Key Management System に保存され、サービスの一部を提供するためにManaged Servicesで管理される必要があります。

## エンドポイントの検出と応答

[!DNL CrowdStrike Falcon]を使用すると、Adobe内のすべてのエンドポイント（サーバを含む）に、軽量で次世代のエンドポイント検出および応答 (EDR) エージェントがインストールされます。 EDR エージェントは、Adobeのデータとシステムをリアルタイムで継続的に監視および収集することで保護し、迅速な脅威の特定と対応を可能にします。

## 侵入テスト

Managed Servicesは、標準のアプリケーションを使用して、Adobe Commerceクラウドシステムの定期的な侵入テストを実施しています。 お客様は、カスタマイズされたアプリケーションの侵入テストを担当します。

## 支払いゲートウェイ

Adobe Commerceでは、クレジットカードのデータが消費者のブラウザーから支払いゲートウェイに直接渡される場合、支払いゲートウェイ統合が必要です。 Adobe Commerce Pro プランのManaged Services環境では、カードデータを使用できません。 e コマースアプリケーションによるトランザクションに対するアクションは、ゲートウェイからのトランザクションへの参照を使用して実行されます。

## Adobe Commerceアプリケーション

Adobeは、コアアプリケーションコードを定期的にテストして、セキュリティの脆弱性を調べます。 お客様には、不具合やセキュリティ上の問題に対するパッチが提供されます。 製品セキュリティチームは、OWASP アプリケーションのセキュリティガイドラインに従って、Adobe Commerce製品を検証します。 いくつかのセキュリティ脆弱性評価ツールと外部ベンダーを使用して、コンプライアンスのテストと検証を行います。 セキュリティツールには次のものが含まれます。

- Veracode 静的および動的スキャン
- RIPS ソースコードスキャン
- Trustwave と Alert Logic の脆弱性スキャンサービス
- Burp Suite Pro
- OWASPZAP
- andSqlMap

フルコードベースでは、これらのツールが隔週でスキャンされます。 お客様には、直接の電子メール、アプリケーションでの通知、および [セキュリティセンター](https://helpx.adobe.com/security.html).

お客様は、PCI ガイドラインに従って、リリース後 30 日以内に、これらのパッチがカスタマイズされたアプリケーションに適用されるようにする必要があります。 Adobeには [セキュリティスキャンツール](https://docs.magento.com/user-guide/magento/security-scan.html) これにより、マーチャントは定期的にサイトを監視し、既知のセキュリティリスク、マルウェア、および不正アクセスに関する更新を受け取ることができます。 セキュリティスキャンツールは無料のサービスで、任意のバージョンのAdobe Commerceで実行できます。

セキュリティ研究者に脆弱性の特定と報告を促すために、Adobe Commerceには [バグバウンティティプログラム](https://hackerone.com/magento) 内部テストに加えて。 さらに、顧客は、必要に応じて、アプリケーションの完全なソースコードを自分のレビュー用に提供されます。

## 読み取り専用ファイル・システム

すべての実行可能コードは読み取り専用のファイル・システム・イメージにデプロイされ、攻撃に使用できるサーフェスが大幅に削減されます。 このデプロイメントプロセスでは、Squash-FS イメージが作成され、PHP または JavaScript コードをシステムにインジェクトしたり、Adobe Commerceアプリケーションファイルを変更したりする機会が減ります。

## リモートデプロイメント

実行可能コードをManaged Services実稼動環境に取り込む唯一の方法は、プロビジョニングプロセスを通じて実行することです。 プロビジョニングでは、ソースリポジトリからリモートリポジトリにソースコードをプッシュし、デプロイメントプロセスを開始する必要があります。 そのデプロイメントターゲットへのアクセスが制御されるので、デプロイメントターゲットにアクセスできるユーザーを完全に制御できます。 非実稼動環境へのアプリケーションコードのデプロイメントは、すべてお客様が管理します。

## ログ

すべてのAWSアクティビティは、AWS CloudTrail にログインします。 オペレーティング・システム、アプリケーション・サーバ、データベース・ログは本番サーバに保存され、バックアップに保存されます。 ソースコードの変更はすべて Git リポジトリに記録されます。 デプロイメント履歴は、Adobe Commerce [Project Web インターフェイス](https://devdocs.magento.com/cloud/project/projects.html#login). すべてのサポートアクセスが記録され、サポートセッションが記録されます。

詳しくは、 [ログの表示と管理](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/test/log-locations.html) （内） _クラウドガイド_.

## 機密データ

機密データは、消費者の個人情報やManaged Servicesのお客様の機密データをカバーできます。 Adobe Commerce Managed Servicesにとって、機密性の高い顧客および消費者データの保護は重要な義務です。 Managed Servicesのお客様もAdobeのお客様も、個人を特定できる情報に関する法的義務を負います。 アーキテクチャのセキュリティ機能に加えて、機密データの配布とアクセスを制限する他のコントロールもあります。

お客様はデータを所有し、そのデータの場所を管理します。 顧客は、実稼動インスタンスと開発インスタンスが存在する場所を指定します。 また、Commerce を使用するAdobe Commerceレポート環境で使用する場所と、Adobe Commerceレポートアプリケーションが個人を特定できる情報にアクセスできるかどうかも指定します。 実稼動インスタンスは、ほとんどのAWS地域に存在しますが、開発環境とAdobe Commerceレポート環境は、現在、米国または欧州連合のいずれかに存在します。

機密データは Fastly CDN サーバーネットワークを通過する場合がありますが、Fastly ネットワークには保存されません。 Managed Servicesのオファーに含まれるすべてのパートナーは、機密データを確実に保護するための契約上の義務を負います。 Managed Servicesは、機密性の高い顧客データや消費者データを、顧客が指定した場所から移動することはありません。

Managed Servicesのスタッフは、Managed Services製品に含まれるサービスを提供する際に、機密データを含む実稼動システムにアクセスする必要があります。 これらの従業員は、機密性の高い顧客および消費者のデータを保護するための義務についてトレーニングを受けます。 これらのシステムへのアクセスは、アクセスを必要とする従業員に限られます。 これらの従業員は、これらのサービスを提供するのに必要な時間だけアクセス権を持っています。

## EU 一般データ保護規則

EU 一般データ保護規則 (GDPR) は、EU（欧州連合）内の個人の個人情報の収集と処理に関するガイドラインを定めた法的枠組みです。 これらの規制は、サイトのベースに関係なく適用され、EU の訪問者はサイトにアクセスできます。

訪問者は、サイトが収集したデータを通知され、情報収集に対して明示的に同意する必要があります。 サイトが保有する個人データが侵害された場合、サイトは訪問者に通知する必要があります。

サイトを運営するマーチャントまたは企業には、サイトのデータセキュリティを監視する専門のデータ保護担当者が必要です。 訪問者がデータの消去を要求した場合は、データプライバシー担当者（または Web サイト管理チーム）が連絡を取ることができます。

GDPR は、匿名化または偽名化された個人を特定できる情報（名前、人種、生年月日など）を収集するよう求めます。

>[!NOTE]
>
>このページでは、GDPR に関して考慮すべき事項の一般的な概要を示します。 詳しくは、 _[セキュリティおよびコンプライアンスガイド](../../../security-and-compliance/privacy/gdpr.md)_ Adobe Commerceが個人情報を保存する方法の詳細については、を参照してください。 ビジネスが法的義務をどのように遵守すべきかを判断するには、弁護士に相談するか、 [公式テキスト](https://eur-lex.europa.eu/eli/reg/2016/679/oj).

## バックアップ

過去 24 時間の操作の間、バックアップは 1 時間ごとに実行されます。 24 時間の後、バックアップは、AWS EBS Snapshot サービスを使用してスケジュールに従って保持されます。 詳しくは、 [保持ポリシー](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/architecture/pro-architecture.html#retention-policy) （内） _クラウドガイド_.

このサービスは、冗長ストレージ上に独立したバックアップを作成します。 EBS ボリュームは暗号化されるので、バックアップも暗号化されます。 また、Managed Servicesは、お客様からリクエストされた方法でオンデマンドでバックアップを実行します。

Managed Servicesのバックアップ/リカバリアプローチでは、フルシステムバックアップと組み合わせた高可用性アーキテクチャを使用します。 各プロジェクトは、3 つの異なるAWSのアベイラビリティゾーンをまたいで、すべてのデータ、コード、アセットを複製します。各ゾーンは別々のデータセンターを持ちます。

詳しくは、 [スナップショットとバックアップの管理](https://experienceleague.adobe.com/docs/commerce-cloud-service/user-guide/develop/storage/snapshots.html) （内） _クラウドガイド_.
